---
title: Secret management commands
description: Securely store and manage API tokens with npmctl
---

Secret management commands allow you to securely store Cloudflare API tokens in your operating system's keyring, avoiding the need to pass them on the command line or in environment variables.

## npmctl cf-token-set

Store a Cloudflare API token securely in the OS keyring.

```bash
npmctl cf-token-set
```

You'll be prompted to enter the token securely (input is hidden) and confirm it.

### Parameters

<ParamField path="--token" type="string">
  Cloudflare API token. If not provided via this flag, you'll be prompted to enter it interactively with hidden input and confirmation.
</ParamField>

### Examples

```bash
# Interactive (recommended)
npmctl cf-token-set
# You'll be prompted:
# Token: [hidden input]
# Repeat for confirmation: [hidden input]

# Non-interactive (less secure)
npmctl cf-token-set --token your_cloudflare_token
```

### What happens

The token is stored in your OS keyring:
- **Linux**: Secret Service (GNOME Keyring, KWallet) or fallback keyring
- **macOS**: Keychain
- **Windows**: Windows Credential Manager

Once stored, certificate commands will automatically use this token without needing the `--cloudflare-api-token` flag.

## npmctl cf-token-delete

Delete the stored Cloudflare API token from the OS keyring.

```bash
npmctl cf-token-delete
```

This permanently removes the token from secure storage.

### Example output

```
Cloudflare token deleted from OS keyring.
```

## npmctl cf-token-status

Check whether a Cloudflare API token is stored in the keyring.

```bash
npmctl cf-token-status
```

### Example output

If a token is stored:
```
Cloudflare token is stored in keyring.
```

If no token is stored:
```
No Cloudflare token stored in keyring.
```

### Use cases

Use this to verify that your token is properly stored before running certificate commands.

## Token priority

When running certificate commands, npmctl looks for the Cloudflare API token in this order:

1. `--cloudflare-api-token` command line flag
2. `CLOUDFLARE_API_TOKEN` environment variable
3. Stored token in OS keyring (via `npmctl cf-token-set`)

<Tip>
  Using the OS keyring (`cf-token-set`) is the most secure method as it avoids exposing tokens in command history or environment variables.
</Tip>
